# SOP: Updating Call Scripts

## What is a call script?

A call script is the talk track an SDR sees when they select a specific **role + trigger** combination in the Call Guide tab. Each script has content customized for all three verticals (Hospitality, Grocery, Healthcare).

## When to use this SOP

- You want to add a script for a role + trigger combination that doesn't exist yet
- You want to update or improve an existing script
- You want to add a new trigger or vertical-specific language

## What you'll need to provide

A complete script has 6 sections. All 6 are required for a new script.

### 1. Intro (one version per vertical)

The opening line the SDR says after "Hey [name], this is [rep] from Ottimate."

**Requirements:**
- 15-20 seconds when spoken aloud
- Opens with the prospect's world and pain, not Ottimate features
- References the trigger naturally
- One version each for Hospitality, Grocery, and Healthcare

**Example (Controller + Staff Turnover, Hospitality):**
> "We work with hotel groups like Pacifica who were dealing with AP staff leaving and new hires taking months to get up to speed. I wanted to see if that's something you're running into."

### 2. Discovery questions (3-5 questions)

Open-ended questions designed to uncover the pain behind the trigger.

**Requirements:**
- Open-ended, not yes/no
- Directly connected to the trigger
- Can be universal (same for all verticals) or vertical-specific

**Example:**
> "Walk me through what happens when an invoice comes in at one of your properties. How much of that is still manual?"

### 3. Listen-for signals

Keywords or phrases that confirm the trigger is real. The SDR watches for these during the conversation.

**Example:**
> "behind on close, overtime, backlogs, training new hires, manual entry, tribal knowledge"

### 4. Recap

A 1-2 sentence transition from discovery to pitch. Summarizes what the SDR heard.

**Example:**
> "So it sounds like [insert pain points they mentioned]. That's exactly what we help with."

### 5. Pitch points (one set per vertical)

2-4 proof points or features, customized per vertical. Lead with outcomes, not features.

**Requirements:**
- One set each for Hospitality, Grocery, and Healthcare
- Reference real customer results when possible
- Outcomes first, features second

**Example (Hospitality):**
> "We work with groups like Pacifica Hotels who process 7,000+ invoices a month. Their AP team went from spending most of their day on manual entry to only handling exceptions."

### 6. Close

The ask for next steps. Can be universal or per-vertical.

**Example:**
> "Would it make sense to set up 15 minutes later this week to show you how this works for hotel groups like yours?"

## Valid options

**Roles:** Controller, CFO, AP Manager/Clerk, Owner/GM/Ops

**Triggers:**
- No known trigger (generic/discovery)
- Staff loss / AP turnover
- Month-end close breaking point
- Audit / compliance pressure
- Vendor payment friction / margin leakage
- Frustration with current solution
- Growth event
- New financial leader

**Verticals:** Hospitality, Grocery, Healthcare

## How to prepare a new script using an AI tool

Copy the prompt below into Claude, Gemini, or ChatGPT. Replace the bracketed fields before running it.

---

**Prompt to copy:**

```
I need to write an SDR cold call script for Ottimate, an AI-powered AP automation
platform. Ottimate serves three verticals: hospitality, grocery, and healthcare.

Write a script for this combination:
- Role: [ROLE - e.g., Controller]
- Trigger: [TRIGGER - e.g., Staff loss / AP turnover]

The script needs these sections. Write each one separately:

1. INTRO (one version for each vertical: hospitality, grocery, healthcare)
   - 15-20 seconds max
   - Open with the prospect's world and pain, not Ottimate features
   - Sound like a real person, not a marketing deck
   - Reference the specific trigger naturally

2. DISCOVERY QUESTIONS (3-5 questions)
   - Open-ended, not yes/no
   - Designed to uncover the pain behind the trigger
   - If the questions differ by vertical, write separate sets for each

3. LISTEN-FOR SIGNALS
   - Keywords or phrases the SDR should listen for that confirm the trigger is real

4. RECAP
   - A 1-2 sentence bridge from discovery to pitch
   - Include a placeholder like [insert pain points they mentioned]

5. PITCH POINTS (one set for each vertical: hospitality, grocery, healthcare)
   - 2-4 bullet points per vertical
   - Lead with outcomes and proof, not features
   - Reference real customer results when possible (e.g., "groups like Pacifica Hotels
     who process 7,000+ invoices a month")
   - Hospitality differentiators: USALI compliance, multi-property GL mapping,
     automated approval routing across locations
   - Grocery differentiators: Item Validation, DSD handling, cost file matching,
     vendor overcharge detection
   - Healthcare differentiators: PHI redaction, HIPAA compliance, multi-entity
     support for senior living and DSO groups

6. CLOSE
   - A soft ask for next steps
   - If the close differs by vertical, write separate versions

Constraints:
- No em dashes. Use commas, periods, or rewrite.
- Write like a human having a conversation, not reading from a script.
- Triggers first, product second. Every line should connect to the prospect's situation.
- Keep it brief. This is a cold call, not a presentation.
```
